## 변경 내역 

- 이전: ultrathink를 입력해야 깊은 사고 모드가 켜짐
- 현재: 아무것도 입력하지 않아도 깊은 사고 모드가 항상 켜져 있음

## (선택) 사고 토큰 설정 바꾸는 방법

클로드 코드의 설정 파일(settings.json)에서 간단하게 바꿀 수 있어요.

설정 파일 위치

- macOS / Linux: ~/.claude/settings.json
- Windows: C:\Users\사용자이름\.claude\settings.json

사고 토큰을 최대(63,999)로 늘리고 싶은 경우:

```json
{
  "env": {
    "MAX_THINKING_TOKENS": "63999"
  }
}
```

복잡한 시스템 설계, 여러 파일에 걸친 대규모 작업 등에서 유용해요. 다만 토큰이 늘어나는 만큼 비용과 응답 시간도 함께 증가한다는 점 참고해주세요.

사고 토큰을 줄이고 싶은 경우 (예: 10,000):

```json
{
  "env": {
    "MAX_THINKING_TOKENS": "10000"
  }
}
```
간단한 작업 위주로 사용하시는 분들은 줄여서 비용을 절약할 수 있어요.

확장 사고를 완전히 끄고 싶은 경우:

```json
{
  "env": {
    "MAX_THINKING_TOKENS": "0"
  }
}
```
💡 이미 settings.json에 다른 설정이 있다면 기존 내용은 그대로 두고 "env" 부분만 추가하면 돼요. 예를 들어 이런 형태가 됩니다:

```json
{
  "permissions": {
    "allow": ["Read", "Write"]
  },
  "env": {
    "MAX_THINKING_TOKENS": "63999"
  }
}
```
설정 후 클로드 코드를 새로 실행하면 적용됩니다.

## 확장 사고 모드

**특정 키워드로 Claude에게 더 많은 사고 시간과 토큰을 할당하는 기능**

"`think`" 키워드를 사용하면 Claude가 문제를 더 깊이 분석하고 대안을 철저히 평가합니다.

| 레벨 | 명령어 | 사고 시간 | 용도 |
| --- | --- | --- | --- |
| **1단계** | `think` | 기본 | 일반적인 복잡한 문제 |
| **2단계** | `think hard` | 중간 | 더 신중한 분석 필요 |
| **3단계** | `think harder` | 많음 | 매우 복잡한 문제 |
| **4단계** | `ultrathink` | 최대 | 극도로 복잡한 문제 |

### 선택 기준

- **단순 작업**: 일반 모드
- **복잡한 계획**: think
- **깊은 분석**: think hard
- **시스템 레벨**: think harder / ultrathink

## 스크린샷 활용

`코드 작성 → 스크린샷 → 반복 개선 워크플로우`

**Claude에게 "눈"을 주어 자신의 작업 결과를 보고 개선하게 하는 방법**

사람처럼 결과물을 보면서 반복 개선하면 훨씬 더 좋은 결과를 얻을 수 있습니다.

### 4단계 프로세스

1. 스크린샷 기능 제공
    - Puppeteer MCP 서버 설치
    - 또는 수동으로 이미지 첨부
2. 목표 디자인 제시
    
    `claude (이미지 첨부) "이 디자인대로 구현해줘"`
    
3. 자동 반복 개선
    
    `claude "구현 → 스크린샷 → 비교 → 수정을 반복해서 디자인과 일치시켜줘"`
    
4. 커밋
    
    `claude "완성됐으면 커밋해줘"`
    

### 반복의 효과

`1차: 70점 → 2차: 85점 → 3차: 95점`

사람처럼 Claude도 반복하면 훨씬 좋아집니다.

- 결과를 볼 수 있게 해주기
- 목표 이미지 제공
- 2-3번 반복으로 큰 개선
- 도구가 핵심
